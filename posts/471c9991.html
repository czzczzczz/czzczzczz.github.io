32 位程序和 64 位程序的辨析

在 PE 的 StudyPE 工具，把软件拖进去即可识别。

32 位软件只能使用× 32dbg 或者 OD 调试，64 位软件只能使用× 64dbg 调试

某本地登录器软件的爆破

破解流程 : 查信息（看是 32 还是 64 位）-> 找内存关键数据-> 测试

32 位，拖到 32dbg 调试，点击左上角小三角运行

问题: 检测我们的密码对不对，原理是 软件算出一个密码 和 我们 输入的密码做比较
这些数据存放在哪里: 内存里

内存布局（去内存找）——右键——搜索匹配特征——双击进入内存窗口——没有可疑信息——引用——继续试——发现可疑信息——选中右键——复制——选中的行——尝试登录

软件标题的修改法(通杀方法一)

  1. 查信息 → 32 位
  2. 拖进 OD 调试，左上角小三角运行程序
  3. 找标题数据：在左下角内存区——右键——查找——二进制字串——输入标题（可能乱码）——确定——若搜索到的是乱码——内存区右键——文本——选择其他的编码集
  4. 设置新标题：选中标题——右键——二进制——编辑——输入新标题（标题长度保持不变，不然程序会崩溃）
  5. 保存：选中更改的标题（选多不选少）——右键——复制到可执行文件——对着弹出的窗口右键——保存文件

软件标题的修改法(通杀方法二)

数据不仅放在内存中，而且有特定的内存地址标号去标识

  1. 查信息 → 32 位
  2. 拖进 OD 调试，左上角小三角运行程序
  3. 找标题数据：在左下角内存区——右键——查找——二进制字串(CTRL + B)——输入标题（可能乱码）——确定——没有找到——换字符集——还是没有
  4. 使用CE查找
    1. 左上角小电脑——选择进程——打开
    2. 数值类型——字符型——输入标题文本
    3. 可写——点击在点击——变为正方形
    4. 首次扫描——没找到——新的扫描——点击UTF-16——首次扫描——找到了
    5. 双击——直接在CE中修改（标题长度保持不变，不然程序会崩溃）
    6. 保存（CE不具备此功能）
  5. 内存区右键——转到——表达式（CTRL + G）——输入CE中查到的内存地址——确定
  6. 右键——文本——更换到CE中显示的字符集
  7. 保存：选中更改的标题（选多不选少）——右键——复制到可执行文件——对着弹出的窗口右键——保存文件

OD 的问题在搜索的时候搜不到，借助了 CE搜，得到数据的标号(内存地址)，在 OD 里转到这个地址，保存文件的操作

软件标题的修改法(通杀方法三 64位程序标题)

  1. 查信息 → 64 位
  2. 拖进 64dbg 调试，左上角小三角运行程序
  3. 内存布局——右键——搜索匹配特征（CTRL + B）——输入标题搜索——没找到——换字符集在搜索——找到
  4. 内存布局——右键——文本——其他字符集尝试——找到标题——选中右键——二进制编辑——编辑（CTRL + E）
  5. 输入新标题——确定——发现无法修改
  6. 引用——选择令一个内存地址
  7. 找到标题——选中右键——二进制编辑——编辑（CTRL + E）
  8. 输入新标题——确定——发现可以修改
  9. 保存：点击上方创可贴——修补文件——请名字（加.exe后缀）——保存

64位程序只能使用64dbg，在搜索结果中进行测试，找到可修改的那个地址，最后用创可贴保存文件

NOP 大法去除软件验证

NOP:空指令(汇编语言 编程语言)，什么都不做
思路:找到判断的关键代码处，直接修改为空指令，就不去判断了，就登录成功了
找关键代码方法:字符串查找法

  1. 查信息 → 64 位
  2. 发现点击登录后弹窗有字符串”Try”
  3. 汇编区——右键——搜索——所有用户模块——字符串——输入“Try”——发现有多个内存地址——挨个试
  4. 选中第一个内存地址——F2下断点——回到软件点登录——发现软件断下来了
  5. 点击CPU——断的位置就显示出来了——它旁边就是关键区域——在关键区域上面，看跳转（蓝线或红线）
  6. 代码是一行一行往下执行的——想办法跳过这个区域（既不弹这个错误窗口）
  7. 选中跳转——右键——二进制——用NOP填充——确定

什么是软件保护壳

刚编写出来的软件是没有保护壳的，壳是软件编写出来后，软件作者之后加的
软件加壳可以干什么:保护软件不被轻易的修改
保护壳分类:压缩壳(UPX、ASPACK)、加密壳(ASProtect、Armadilo[穿山甲]、ExeCryptor、Themida)、虚拟机保护软件(VMProtect)，是一种软件

了解什么是内存地址

  1. 内存地址:一个数，可以理解为标号，就像身份证号一样，每个人都标记起来
  2. 程序构成:代码 +数据
  3. 代码 +数据都被内存地址所标记
  4. 代码也是内存数据的一种，由于它能被执行，所以我们一般称之为“代码”，而不直接称之为数据
  5. 回顾跳转、断点、字符串

总结

  1. 破解流程:查壳->拖到 x32-64dbg/0D，来到入口点(真/假)，此时程序内存区域在主模块领空
  2. 如果要破解标题，就在内存布局搜索字符串(不同编码），实在不行去CE搜
  3. 如果要对某些功能进行拦截监测，就在 Windows API 函数上下断点
  4. 如果不知道什么函数，就批量打断点
  5. 如果想知道程序干了什么，用进程信息拦截器，火绒剑
  6. 关键位置的破解方法:程序块头部ret、nop、，根据箭头流程修改 jmp 强制跳转

寻址方法—立即寻址

寻址:寻址地址，寻找谁的地址?数据的地址
回顾之前 VB 程序版本号的逆向分析，版本号其实就是数据，找到这个数据来源，更改，就实现了更改程序的版本号
同理:找到软件时间验证的存储地址，直接更改，就更改了软件的使用时间限制
7个方法：立即寻址、寄存器寻址、直接寻址、寄存器间接寻址，寄存器相对寻址、基址变址寻址、基址变址相对寻址

立即寻址:数据直接存放在代码中，汇编中整数常量称作立即数
技巧:寻址看 mov

形式：右边直接就是一个数

寄存器寻址

操作数的值存储在寄存器的寻址方式称作寄存器寻址
寄存器包括通用寄存器、段寄存器

Call 函数 函数会返回一些数据:返回值–>一般是传递给 eax,的

直接寻址

操作数值在内存中，机器码中存储16位段内偏移的寻址方式称作直接寻址

取出内存地址的数据：[内存地址]

[] = 取出数据

中括号里面永远都是内存地址

寄存器间接寻址

操作数值在内存中，段内偏移存储在寄存器中的寻址方式称作寄存器间接寻址

寄存器相对寻址

操作数值在内存中，段内偏移存储由[寄存器 +立即数]计算得来的寻址方式称作寄存器相对寻址

拓展：

  • 快速的跑到某行：右键——在此设置EIP
  • EIP ：当前指令执行的位置，可以过掉一些调试的检测或者校验

基址变址寻址

操作数值在内存中，段内偏移由[寄存器+变址寄存器]计算得来的寻址方式称作基址变址寻址

基址变址相对寻址

操作数值在内存中，段内偏移由[基址寄存器+变址寄存器 + 偏移常量]计算得来的寻址方式称作基址变址寻址

段的标记

2个常见寄存器和对应的含义
Ss:Stacksegment，栈段寄存器
当前堆栈的底部地址，即存放堆栈段的段基址
DS:Data Segment，数据段寄存器
当前程序使用的数据所存放段的最低地址，即存放数据段的段基址

总结：
常见两种情形，ebp，esp用ss段标记，其他通用寄存器都是 ds段

汇编指令专题讲解-数据传送指令

  1. Mov 指令
    指定传送数据的长度:
    Byte 字节 word 双字节 dword 四字节(double 双倍的)【32 位程序内存地址都是四字节】
    ptr:pointer 的缩写，翻译为指针，其实就是内存地址

  2. 交换指令：xchg
    情形:寄存器与寄存器之间对换数据、寄存器与存储器
    效率:xchg 优于 mov，因为 xchg,使用了内部暂存器

  3. 堆栈操作指令
    进栈:(push reg)
    Sub sp,2
    Mov [sp],reg
    出栈(pop reg)
    Mov reg,[sp]
    Add sp,2
    堆栈窗口:显示的是数据的临时存取情况
    保存所有寄存器环境【过环境上下文的检测】: 16 位:pusha/popa 32 位:pushad/popad

  4. 标志寄存器传送指令

    低8位传送:Lahf和 sahf l:load s:stove 把标志寄存器低八位传送给 ah 把ah 寄存器中的值传送给标志寄存器低八位
    16位传送:pushf 和 popf 把标志寄存器的值直接压入栈中 把栈中的值弹出到标志寄存器上

  5. 地址传送指令
    作用:将存储单元的地址传送到指定的寄存器
    有效地址传送指令:lea
    例子:
    Lea bx,[bx+si] //相当于bx =bx+ si 给的是后面的地址，不是值
    Mov bx,[bx+si] 给的是后面的地址中的值

  6. 指针传送指令:Ids les

汇编指令专题讲解-算术运算指令

  1. 加法
    add ： add eax，3
    adc ： adc eax,3 = add eax,3+CF
    inc : 自加1（对CF无影响）
  2. 减法
    sub ：sub eax，3
    sbb ：sbb eax,3 = sub eax,3-CF
    dec : dec eax，自减1
  3. 求补指令
    neg ax ：若 ax = 0，则 CF = 0；若ax != 0，CF = 1
  4. 比较指令
    cmp eax,b 不存结果，只影响标志位
    作用：一般后面跟一个条件跳转(转移)指令，根据比较结果转向不同的程序分支
  5. 乘法指令
    无符号乘法：mul
    有符号乘法：imul
  6. 除法指令
    无符号除法 ： div
    有符号除法 ： idiv

汇编指令专题讲解-逻辑运算指令

  1. 按位非 not
  2. 按位或 or
  3. 按位与 and test（不存结果）
    应用：把某些数据清零 （and eax，0）
  4. 按位异或 xor （相同是0，不相同是1）
    应用： 寄存器请0 （xor eax，eax）
