基于密码的登录中的漏洞

用户名枚举

用户名枚举是指攻击者能够观察网站行为的变化，以确定给定的用户名是否有效。

用户名枚举通常发生在登录页面上，例如，当您输入了有效的用户名但密码不正确时，或者在注册表单上输入了已采用的用户名。这大大减少了暴力破解登录所需的时间和精力，因为攻击者能够快速生成有效用户名的候选列表。

在尝试暴力破解登录页面时，应特别注意以下方面的任何差异：

  • 状态代码：在暴力攻击期间，对于绝大多数猜测，返回的HTTP状态代码可能相同，因为大多数猜测都是错误的。如果猜测返回不同的状态代码，则强烈表明用户名是正确的。无论结果如何，网站始终返回相同的状态代码是最佳做法，但并不总是遵循这种做法。
  • 错误消息：有时返回的错误消息会有所不同，具体取决于用户名和密码是否都不正确或只有密码不正确。网站在这两种情况下都使用相同的通用消息是最佳做法，但有时会出现小的键入错误。仅一个字符不合适即可使两条消息截然不同，即使在该字符在呈现的页面上不可见的情况下也是如此。
  • 响应时间：如果大多数请求都是以相似的响应时间处理的，则任何偏离此时间的请求都表明幕后发生了一些不同的事情。这是猜测的用户名可能正确的另一个迹象。例如，如果用户名有效，网站可能仅检查密码是否正确。此额外步骤可能会导致响应时间略有增加。这可能很微妙，但攻击者可以通过输入网站需要明显更长的处理时间过长的密码来使这种延迟更加明显。

1.通过不同的响应枚举用户名

  1. 运行Burp后，调查登录页面并提交无效的用户名和密码。

  2. 在 Burp 中，转到代理> HTTP 历史记录并找到请求。将其发送给Burp Intruder。POST /login

  3. 在“打嗝入侵者”中，转到“位置”选项卡。确保选择了狙击手攻击类型。

  4. 单击清除 § 以删除任何自动分配的有效负载位置。突出显示参数的值，然后单击添加 § 以将其设置为负载位置。此位置将由两个符号指示，例如：.暂时将密码保留为任何静态值。username``§``username=§invalid-username§

  5. 在“负载”选项卡上，确保选中“简单列表负载类型”。

  6. 在“有效负载选项”下，粘贴候选用户名列表。最后，单击“开始攻击”。攻击将在新窗口中打开。

  7. 攻击完成后，在“结果”选项卡上，检查“长度”列。您可以单击列标题对结果进行排序。请注意，其中一个条目比其他条目长。将此有效负载的响应与其他响应进行比较。请注意，其他响应包含该消息，但此响应显示 。记下“负载”列中的用户名。Invalid username``Incorrect password

  8. 关闭攻击并返回“位置”选项卡。点按“清除”，然后将参数更改为您刚刚标识的用户名。将负载位置添加到参数。结果应如下所示：username``password

  9. 在“负载”选项卡上，清除用户名列表并将其替换为候选密码列表。单击“开始攻击”。

  10. 攻击结束后，查看“状态”列。请注意，每个请求都收到一个带有状态代码的响应，但有一个请求获得了响应。这表明登录尝试成功 - 在“有效负载”列中记下密码。200``302

  11. 使用您标识的用户名和密码登录，并访问用户帐户页面以解决实验室问题。

    注意

    也可以使用单个集束炸弹攻击暴力登录。但是，如果可能的话，首先枚举有效的用户名通常要有效得多。

2.通过微妙不同的响应进行用户名枚举

  1. 运行 Burp 后，提交无效的用户名和密码。将请求发送到 Burp 入侵者，并向参数添加有效负载位置。POST /login``username

  2. 在“有效负载”选项卡上，确保选中“简单列表有效负载类型”，并添加候选用户名列表。

  3. 在“选项”选项卡上的“Grep - 提取”下，单击“添加”。在出现的对话框中，向下滚动响应，直到找到错误消息。使用鼠标突出显示邮件的文本内容。其他设置将自动调整。单击“确定”，然后开始攻击。Invalid username or password.

  4. 攻击完成后，请注意，还有一列包含您提取的错误消息。使用此列对结果进行排序，以注意到其中一个结果略有不同。

  5. 仔细查看此响应，请注意它在错误消息中包含拼写错误 - 而不是句号/句点，而是尾随空格。记下此用户名。

  6. 关闭攻击并返回“位置”选项卡。插入您刚刚标识的用户名，并向参数添加有效负载位置：password

  7. 在“有效负载”选项卡上，清除用户名列表并将其替换为密码列表。开始攻击。

  8. 攻击完成后，请注意其中一个请求收到了响应。记下此密码。302

  9. 使用您标识的用户名和密码登录，并访问用户帐户页面以解决实验室问题。

3.通过响应时序枚举用户名

  1. 运行 Burp 后，提交无效的用户名和密码，然后将请求发送到 Burp Repeater。尝试不同的用户名和密码。请注意，如果您尝试登录次数过多，您的 IP 将被阻止。POST /login

  2. 确定标头是否受支持，这允许您欺骗 IP 地址并绕过基于 IP 的暴力破解保护。X-Forwarded-For

  3. 继续尝试使用用户名和密码。特别注意响应时间。请注意，当用户名无效时，响应时间大致相同。但是，当您输入有效的用户名（您自己的用户名）时，响应时间会根据您输入的密码的长度而增加。

  4. 将此请求发送给Burp Intruder，然后选择Pitchfork的攻击类型。清除默认负载位置并添加标头。X-Forwarded-For

  5. 为标头和参数添加负载位置。将密码设置为很长的字符串（应为大约100个字符）。X-Forwarded-For``username

  6. 在“负载”选项卡上，选择负载集 1。选择“数字”负载类型。输入范围 1 - 100 并将步长设置为 1。将最大分数位数设置为 0。这将用于欺骗您的IP。

  7. 选择有效负载集 2 并添加用户名列表。开始攻击。

  8. 攻击结束后，在对话框顶部单击“列”，然后选择“收到的响应”和“响应已完成”选项。这两列现在显示在结果表中。

  9. 请注意，其中一个响应时间明显长于其他响应时间。重复此请求几次，以确保它始终需要更长的时间，然后记下此用户名。

  10. 为同一请求创建新的 Burp 入侵者攻击。再次添加标头并向其添加有效负载位置。插入您刚刚标识的用户名，并向参数添加有效负载位置。X-Forwarded-For``password

  11. 在负载选项卡上，添加负载集 1 中的数字列表，并将密码列表添加到负载集 2。开始攻击。

  12. 攻击完成后，找到具有状态的响应。记下此密码。302

  13. 使用您标识的用户名和密码登录，并访问用户帐户页面以解决实验室问题。

    注意

    也可以使用单个集束炸弹攻击暴力登录。但是，如果可能的话，首先枚举有效的用户名通常要有效得多。

SQL 注入

SQL 注入示例 在不同情况下会出现各种各样的SQL注入漏洞，攻击和技术。一些常见的 SQL 注入示例包括：

检索隐藏数据，您可以在其中修改 SQL 查询以返回其他结果。 颠覆应用程序逻辑，您可以在其中更改查询以干扰应用程序的逻辑。 UNION 攻击，您可以从不同的数据库表中检索数据。 检查数据库，您可以在其中提取有关数据库的版本和结构的信息。 盲 SQL 注入，其中您控制的查询的结果不会在应用程序的响应中返回

检索隐藏数据

考虑一个显示不同类别产品的购物应用程序。当用户单击“礼品”类别时，其浏览器会请求 URL：

https://insecure-website.com/products?category=Gifts 这会导致应用程序进行 SQL 查询以从数据库中检索相关产品的详细信息：

SELECT * FROM products WHERE category = 'Gifts' AND released = 1 此 SQL 查询要求数据库返回：

所有详细信息 （*） 从产品表中 其中类别为“礼品” 并释放是 1。 该限制用于隐藏未发布的产品。对于未发布的产品，大概是 .released = 1released = 0

应用程序不会对 SQL 注入攻击实施任何防御措施，因此攻击者可以构造如下攻击：

https://insecure-website.com/products?category=Gifts'-- 这将生成 SQL 查询：

SELECT * FROM products WHERE category = 'Gifts'--' AND released = 1 这里的关键是双破折号序列是 SQL 中的注释指示器，这意味着查询的其余部分被解释为注释。这将有效地删除查询的其余部分，因此它不再包含 。这意味着将显示所有产品，包括未发布的产品。--AND released = 1

更进一步，攻击者可以使应用程序显示任何类别中的所有产品，包括他们不知道的类别：

https://insecure-website.com/products?category=Gifts'+OR+1=1-- 这将生成 SQL 查询：

SELECT * FROM products WHERE category = 'Gifts' OR 1=1--' AND released = 1 修改后的查询将返回类别为“礼品”或“1”等于 1“的所有项目。由于 始终为 true，因此查询将返回所有项目。1=1 https://portswigger.net/web-security/sql-injection/lab-retrieve-hidden-data

颠覆应用逻辑

考虑一个允许用户使用用户名和密码登录的应用程序。如果用户提交了用户名和密码，应用程序将通过执行以下 SQL 查询来检查凭据：wienerbluecheese

SELECT * FROM users WHERE username = 'wiener' AND password = 'bluecheese' 如果查询返回用户的详细信息，则登录成功。否则，它将被拒绝。

在这里，攻击者可以以任何没有密码的用户身份登录，只需使用 SQL 注释序列从查询子句中删除密码检查即可。例如，提交用户名和空白密码会导致以下查询：--WHEREadministrator'--

SELECT * FROM users WHERE username = 'administrator'--' AND password = '' 此查询返回用户名所在的用户，并成功以该用户身份登录攻击者。administrator https://portswigger.net/web-security/sql-injection/lab-login-bypass

从其他数据库表中检索数据

如果在应用程序的响应中返回 SQL 查询的结果，攻击者可以利用 SQL 注入漏洞从数据库中的其他表中检索数据。这是使用关键字完成的，该关键字允许您执行其他查询并将结果追加到原始查询。UNIONSELECT

例如，如果应用程序执行以下包含用户输入“Gifts”的查询：

SELECT name, description FROM products WHERE category = 'Gifts' 然后攻击者可以提交输入：

' UNION SELECT username, password FROM users-- 这将导致应用程序返回所有用户名和密码以及产品的名称和描述。

CDN检查

一、什么是CDN？

CDN的全称是Content Delivery Network，翻译成中文就是内容分发网络。CDN依靠部署在各地的边缘服务器，通过中心平台的负载均衡、内容分发、调度等功能模块，使用户就近获取内容，降低网络延迟，提高访问速度。简单来讲，CDN是用来进行加速的，它可以让用户更快获得所需的数据。举个例子，某个网站的服务器在北京，如果深圳的用户想要获取服务器上的数据，就需要跨越很远的距离，由于传输速度和路由转发等因素，就会导致访问速度非常缓慢。但如果我们在深圳建立一个CDN服务器，上面缓存了一些服务器数据，那么深圳用户只需要访问这个CDN的服务器就能获取相关的内容，这样速度就提升了很多。但在安全测试过 程中，若目标存在 CDN 服务，将会影响到后续的安全测试过程。

二、如何判断目标存在CDN服务?

利用多节点技术进行请求返回判断 https://ping.chinaz.com/ 使用超级ping工具进行测试，若全国ip测试都一样，即没有开启cdn，不一样则反之。

三、目前常见的CDN绕过技术有哪些?

  1. 利用子域名请求获取真实IP

  2. 利用国外地址请求获取真实IP

  3. 利用第三方接口查询获取真实IP

  4. 利用邮件服务器接口获取真实IP

  5. 利用黑暗引擎搜索特定文件获取真实IP
