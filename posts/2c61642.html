typora 快捷键

下载安装与激活

在 官网 下载1.9.3版

安装时位置不要选择 C 盘，并且最后的运行 typora 一定不要勾选

激活工具在我的网盘-tool-winmm.dll 放在 typora 安装文件夹（如 D:\Typora）

打开 typora，点击帮助-我的许可证，显示已激活，则成功激活。

typora 基本设置

文件-偏好设置

  1. 文件-勾选自动保存
  2. 编辑器—拼写检查—不使用拼写检查
  3. 图像-插入图片时-上传-勾选对网络图像
    上传服务-picgo（app）-下载 app(可能加载慢) -安装 -下面内容的图床设置
  4. markdown-拓展语法全部勾选
  5. 通用-勾选使用国内服务器

常用快捷键

把网络图片转到自己图床：先等待文档中的图片全部加载完毕，然后选择 复制所有图片到…，复制完成后，在 选择上传所有文件

根据 Markdown 最佳实践，一篇文档应该 有且仅有 一个 h1，故此插件从 h2 开始编号。(h1 就是文件名，可以不用写)

ctrl + 1 到 6 对应 1 到 6 级标题

或者使用 ## text 几个#就是几级标题

CTRL 0 对应正文

列表嵌套

无序列表

  • 先输入 - 空格

  • 按 enter 进入下一行自动生成黑点

    • enter 在按 tab 缩进
      shift + enter 进入下一行，但是不会开启新的列表

    按 enter 在按 enter 退出列表

  • 按 enter 后再按 tab＋shift 回到上一层列表

按 enter 在按 enter 退出列表

有序和无序列表

  1. 输入 1.＋空格
  2. enter 进入下一行
    1. tab 进入嵌套
      • 按回车后再按 tab，然后将 2.删除，输入 -＋空格，进入无序嵌套
      • enter
    2. shift + tab 返回上一层嵌套
  3. shift + tab 返回上一层嵌套

两次 enter 退出嵌套

  • 写文章时为了便于分标题等级，可以在左侧大纲右键，设置 大纲视图无折叠。

公式

链接

行内公式：alt + q (似乎不是默认的快捷键，， 可以使用:$你的内容$)

行间公式：alt + w (同上，可以插入公式)

（详见1.7 自定义快捷键）

可以配合软件极度公式使用

识别公式完成后：

  • 如果是行内公式，则直接选择复制 LaTex，（前两个都一样）它自带$符号，所以在 typora 中不用再写了。

  • 如果是行间公式，则直接全选复制，或者点击更多，然后点击 LaTex，这样不含有$符号，然后再 typora 中按 alt+w 公式块，复制进去即可

gif 和图片一样，直接复制粘贴即可

auto=1 参数，这个播放器会默认自动播放音乐。auto=0。这样修改后，音乐将不会在页面加载时自动播放。

图床设置

去不图床 我的图库 (7bu.top)

购买订阅后创建一个 token（注意 token 名称不要有空格）（blog_img3的Token 1750|TEU21610ou3pWtRhEL33cfv2OA8TTx9Z1JIhPNVY）

只保留上传图片即可

获取到 token 后打开 picgo

  • 在插件设置里搜索 lankong 安装完成后如上图设置即可（电脑需有node.js，一直点下一步安装，完成后重启picgo）

​ Server：https://7bu.top 把 ④ 中的 token 换成自己的(Bearer 1750|TEU21610ou3pWtRhEL33cfv2OA8TTx9Z1JIhPNVY)，注意有前缀 Bearer 和空格 (复制 token 时注意前后有没有多余的符号)

​ 确定设为默认图床，(改 picgo 路径), 然后打开 typora 尝试上传图片即可。

  • 或者使用 web插件
    1. 进入插件设置，搜索关键词 web，安装自定义 Web 图床插件（电脑需有node.js）：
    2. 依次填写 API 地址 https://7bu.top/api/v1/upload，POST 参数名为 file，JSON 路径 data.links.url，自定义请求头为{"Authorization": "Bearer TOKEN"}「记得将 TOKEN 换成自己的哦，如{“Authorization”: “Bearer 1750|TEU21610ou3pWtRhEL33cfv2OA8TTx9Z1JIhPNVY”}」
    3. 确定设为默认图床，(改 picgo 路径), 然后打开 typora 尝试上传图片即可。

多个 md 合并

步骤

把需要拼接的 md 文件放入一个文件夹内

  1. 打开命令界面: Windows+R 输入 cmd

  2. 进入 md 文件所在位置: cd+文件夹路径

    1. 或者直接在该文件夹路径内输入cmd回车

  3. 将拼接结果输出到指定路径的 text.md 文件内 type *.md > 指定路径/text.md 【千万 不要指定为想要合成的 md 文件所在文件夹，不然会出现无限合并，注意注意！！】

    • 无需提前建立 text.md 空文件
    • md 文件的拼接顺序需要手动调整
    • 生成上一个文件夹中 index.md : type *.md > …/index.md

出现中文乱码/问号问题解决办法

实操过程我遇到了两个问题：

  1. 生成的 md 文件文字部分是中文乱码
  2. 生成的 md 文件有许多菱形的问号图标

解决办法:

  1. 快捷键打开控制面板 Windows+R，输入 control
  2. “时钟和区域”选项下选择“更改日期、时间或数字格式”，选择“管理”选项卡，选择 更改区域设置，勾选 Beta版：使用Unicode UTF-8提供全球语言支持（U） 后，立即重启电脑
  3. 重新拼接一遍，即可解决

typora 插件使用

下载

目前此方法仅限 Windows 平台。

  1. 下载 插件源码的压缩包，并解压

  2. 进入 Typora 安装路径，找到包含 window.html 的文件夹 A（一般是 Typora/resources/window.html 或者 Typora/resources/app/window.html）

  3. 打开文件夹 A，将源码的 plugin 文件夹粘贴进该文件夹下（参考方法二的图片）

  4. 进入文件夹 A/plugin/bin/，双击运行 install_windows_amd_x64.exe。如果看到下图，说明成功

  5. 验证：重启 Typora，在正文区域点击鼠标右键，弹出右键菜单栏，如果能看到 常用插件 栏目，说明一切顺利

标签页管理

先 CTRL+shift+3 打开文件树

切换标签：ctrl+tab

中英文混排优化

中英文混排时，中文与英文之间、中文与数字之间添加空格。

右下角点击混排优化

根据文档大纲一键生成思维导图

使用方式：右键菜单 -> 常用插件 -> 思维导图-> graph

自动编号

支持编号的组件（皆可临时或永久打开/关闭）：(改变其中之一其他的会自动改变)

  1. 标题
  2. 大纲
  3. TOC（TableofContents ：右侧大纲目录）
  4. 表格
  5. 图片
  6. 代码块

注意：使用自动编号功能时，导出的WORD没有编号，但是导出的HTML和PDF有编号（包括图片编号）

通过注入 CSS 实现此功能，有可能会与你使用的 theme 冲突。

和其他使用 Theme CSS 的实现方式不同，此插件通过修改内置函数，完美解决导出 PDF 后侧边栏没有编号的问题 :)

根据 Markdown 最佳实践，一篇文档应该 有且仅有 一个 h1，故此插件从 h2 开始编号。（h1 就是文件名啦~）

拖动调整表格大小

ctrl+鼠标拖动：修改表格的行高列宽。

调整图片大小

ctrl+鼠标滚轮滚动：调整图片大小。（但是好像不保存）

格式规范检测

使用方式：

  • 方式一：右键菜单 -> 常用插件 -> 自定义插件 -> md 格式规范检测
  • 方式二：点击右上角的小方块
  • 技巧：右键点击右上角表格可快速切换源码和渲染所得，左键点击 op中的定位快速定位到指定行
